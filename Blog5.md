# Blog 5: Events
As one of the goals of the games is to be humorous and random, events that happen during the 1v1 match were decided to be a good addition. In the end the events implemented are different then the ones originally stated. An event manager was implemented that contains a list of available events and calls upon them to activate. They activate 30 seconds after the start of the match. We have a flooding event where the king gets bored and decides he wants his champions wet. The water starts rising and as the response the player gets slower and finds it harder to maneuver. This was implemented by having a shader for water and an object covering the arena to which it is applied to. When the event activates the water objects is slowly raised and as such gives the illusion of the arena being filled with water. Another event is a classic activation of pit traps in the center of the map which means the player now needs to watch his step or will be impaled by spikes. Here in the script instead of manually assigning the traps to a list we get all objects with the trap tag and then activate the traps. Lastly we have a disco event. The pope receives a message from god and his message is to party. A disco ball appears in the sky, music changes and the screen starts to shake.

# Menus
To let the user pause the game and to enhance the felling of the game Main and pause menu were implemented. They yhemselves are simple canvas objects that contain TextMeshPro buttons and text. Each menu has its own script and the buttons are assigned the methods from these scripts. The only difference in implementation is that The main menu has its own scene while the pause menu stops the game time and activates/deactivates the container object.
