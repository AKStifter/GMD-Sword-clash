# Blog 3: First milestone the map and the player animations
When making the game the first thing I thought about was what kind of map would reflect the story of the game. As it is the tournament the most appropriate answer I thought of was an arena. The arena itself is relatively simple. It was mostly made of wall and tile assets found in the unity asset store. That being said it took some time to find the assets that fit the theme and don't look to different from each other. Some things were also impossible to find in the form I wanted them like for example a pit trap with spikes. In this case I found a spike model and texture, and used them for the spikes at the ground of the trap. For the trap walls I used 3d cubes to fashion the trap walls and applied an imported texture. I also added a bar gate as the flooring on which the player walks and which can be lowered with an animation so when the trap is activated he falls in. The philosophy of the arena is to make functional sense as such it is simple so the the player can keep na eye on the enemy, but still has a couple of columns or walls to potentially take shelter from his attacks.

# Player animations, and controls
When deciding on animations one of the things taken into account were which actions the player will be able to perform. As the game needs to be playable on the arcade machine it means it has a limited number of buttons and as such a limited number of actions. That is why it was decided to keep it simple. There is a possibility of jumping, performing attacks both light and heavy (uses the same button), and defending. There are also other player actions like pausing the game or picking up the weapons on the floor but they don't have animations themselves. For the implementation of controls the unity input system was used as it is newer at it helps avoid unnecessary code to accommodate other devices. The player can use multiple weapons so there are variations of attacks for each weapon. There are also variations in player death animations so the game isn't monotone. For certain animations like attack animations, the blending of animations with the help of layers was also used so the the player moves and attacks at the same time it doesn't look weird.
