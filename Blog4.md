# Blog 4: Hit detection
While implementing hit detection I encountered a problem I didn't think about before. It was that the weapon would always deal damage even outside the attack animations. While getting damaged multiple times from one attack was avoided by characters having an invurnability window after being hit, if you walked to the enemy at the wrong angle and touched their weapon you would be wounded or vice versa. To solve this issue all attack animations needed to have animation events that would enable dealing damage during the animation and disable it at the end. For this to work a complementary script was also created so the events could be subscribed to and when activated would go into another layer and toogle a bool that determines if the attack should be accepted when the collision with the weapon is detected.

# Creating enemies and the health system
As it is a tournament game on each level there is only one enemy. While the enemies are standard nav mesh agents, each enemy has somewhat tweaked behavior in terms of attack range, speed and chasing distance. They all of course have their unique attacks and the perform them in random so there is no discernable pattern to which they attack in order to keep things fresh.
The enemy health system isn't seperated from the players. The script used is the same and only thing different is the UI bar that the script uses. Enemies have a UI bar under a canvas object attached to them, while the player has an UI bar in the upper left corner to show their health. 

# Weapon pickups
As the intention was to let the player pickup the desired weapon during the match it was neccesary to make weapon pickups, and a system that would allow the player to pick them up. This also means the player would need to drop one weapon if he picks up another. As the game doesn't have a large number of weapons to choose from we used an illusion of picking a weapon up. Players have a disabled weapon model already attached to them. When picking up a weapon we merely destroy the pickup and activate the weapon. Similarly we do not actually drop the weapon but we spawn it. The weapon pickups were made into prefabs. In the script we keep count on what is the current weapon and if we have a shield as well, and when necessary to drop them we spawn them from the prefabs while also changing the animation tags to make sure all weapons use proper animations. A separate WeaponEntry class was also made to link the equiped weapons to the prefab needed when the droping weapons.
